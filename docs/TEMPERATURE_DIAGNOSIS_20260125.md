# 温度诊断报告（为什么 T=1 不够）

**目标**：用数据说明“logit 尺度偏大 → sigmoid 过于极端 → 连乘分布过尖 → T=1 不够”。

## 1. 实验设置
- checkpoint: `outputs/california_housing/run_20260124_104159/checkpoint.pt`
- 数据集：California Housing 验证集（random_state=0）
- 模型：TransformerTabSeqModel（与训练配置一致）

## 1.5 直觉版解释（先看这段）
- temperature 只是把 logit “缩小或放大”，不是改模型权重。
- logit 很大时，`sigmoid(logit)` 会接近 0/1，概率会“极端”。
- 本项目是“每步概率连乘”，极端概率会被连乘放大，导致区间很窄、PICP 很低。
- 所以 T 需要更大来“降温”，把概率拉回中间态，让区间变宽。

小例子（数字直观）：
- 假设某步 logit=5：
  - T=1 → sigmoid(5)=0.993（几乎 1）
  - T=10 → sigmoid(0.5)=0.622（不再极端）
- 连乘放大（depth=6）：
  - 0.993^6 ≈ 0.958（非常尖）
  - 0.622^6 ≈ 0.056（分布更平）
结论：T 大时，分布更平，区间更宽，PICP 才会上升。

## 6. 结论（用数据解释“为什么 T=1 不够”）
- Logit 的尺度偏大，导致 T=1 时 sigmoid 输出大量接近 0/1 的极端概率。
- 逐步 sigmoid + 连乘会放大这种极端性，使叶子分布过尖，区间过窄。
- T 增大后饱和率下降、叶子分布熵上升，区间变宽，PICP 才上升。
