## 第一部分：两种方法的详细对比

### 第一步：理解问题背景

假设场景：预测房价，取值范围是 0 到 5（百万元）

### 第二步：方法核心对比

两种方法的根本差异体现在以下表格中：

| 对比维度 | 分桶回归（Bin Classification） | 自适应区间预测 |
| --- | --- | --- |
| 任务性质 | 回归→分类 | 直接预测区间 |
| 区间划分 | 预先固定划分（如25类） | 不预先划分 |
| 区间宽度 | 固定（如0.2） | 不固定（自适应） |

| 评估方式 | 命中率（Accuracy） | 命中率 + 区间宽度 |
| 命中判断 | 真实值是否落在预测的桶内 | 真实值是否落在预测区间内 |
| 优点 | 简单直观，易于实现 | 区间宽度自适应，更灵活 |
| 缺点 | 固定宽度无法适应数据特性；小区间命中率低，大区间无意义 | - |

## 第二部分：CDF方法的问题与改进方案

### 第一步：理解CDF方法是如何工作的

累积分布函数（CDF）方法的核心思想：
1. 模型输出概率分布（如25个桶的概率）
2. 从左到右累加概率，找到第一个CDF ≥ 0.05的位置作为下界
3. 继续累加，找到第一个CDF ≥ 0.95的位置作为上界
4. 中间90%的区间就是预测的置信区间

---

### 第二步：CDF方法在三种场景下的表现

下表展示了CDF方法的致命缺陷：

| 场景 | 概率分布特征 | 90%区间起点 | 90%区间终点 | 区间宽度 | 问题分析 |
| --- | --- | --- | --- | --- | --- |
| 场景1：理想尖锐分布 | 单一峰值0.89，集中在桶12 | 桶11 (2.2) | 桶13 (2.6) | 0.4 | ✓ 合理：区间小且精确 |
| 场景2：平缓分布 | 几乎均匀，每个桶约0.04 | 桶1 (0.2) | 桶18 (3.6) | 3.4 | ✗ 不合理：累积了太多低概率区间 |
| 场景3：多峰分布 | 三个峰：0.35, 0.40, 0.15 | 桶8 (1.6) | 桶18 (3.6) | 2 | ✗ 不合理：包含了不连续的多个峰 |

---

### 第三步：详细分析每个场景

**场景1（理想情况）：尖锐分布 ✓**

| 桶编号 | 值域范围 | 概率 | CDF |
| --- | --- | --- | --- |
| 10 | [2.0, 2.2) | 0.005 | 0.005 |
| 11 | [2.2, 2.4) | 0.05 | 0.055 ← 超过0.05，选为下界 |
| 12 | [2.4, 2.6) | 0.89 | 0.945 |
| 13 | [2.6, 2.8) | 0.05 | 0.995 ← 超过0.95，选为上界 |
| 14 | [2.8, 3.0) | 0.005 | 1 |

90%区间：桶11到桶13，宽度仅0.6（3个桶）  
✓ 结论：这是理想情况，区间小且包含了最高概率

---

**场景2（问题场景）：平缓分布 ✗**

| 桶编号 | 概率 | 说明 |
| --- | --- | --- |
| 0 | 0.04 | 低概率 |
| 1 | 0.04 | 低概率 |
| ... | ... | ... |
| 10月14日 | 0.08 | 稍高区域 |
| ... | ... | ... |
| 20-23 | 0.04 | 低概率 |

90%区间：桶1到桶18，宽度3.4（17个桶）  
✗ 问题：为了累积到90%，不得不包含大量低概率(0.04)的区间，这些区间根本不应该在预测范围内！

问题根源：
- 每个桶概率都很小（0.04），需要累积很多桶才能达到0.9
- 这些低概率桶实际上表明"模型不确定"，不应该包含在置信区间内
- 最终得到的区间过宽（3.4），几乎覆盖整个值域，毫无意义

---

**场景3（问题场景）：多峰分布 ✗**

| 桶编号 | 值域范围 | 概率 | 说明 |
| --- | --- | --- | --- |
| 7 | [1.4, 1.6) | 0.03 | 峰1边缘 |
| 8 | [1.6, 1.8) | 0.35 | 峰1中心 |
| 9 | [1.8, 2.0) | 0.03 | 峰1边缘 |
| 10月11日 | - | 0 | 断层区域 |
| 12 | [2.4, 2.6) | 0.4 | 峰2中心 |
| 13-16 | - | 0 | 断层区域 |
| 18 | [3.6, 3.8) | 0.15 | 峰3中心 |

90%区间：桶8到桶18，宽度2.0  
✗ 问题：
- 区间跨越了三个不连续的峰
- 中间有大量概率为0的区域（断层）
- 从物理意义上讲，预测一个 [1.6, 3.8] 的区间是不合理的
  - 模型认为答案可能在1.6-1.8，或者2.4-2.6，或者3.6-3.8
  - 但CDF强行把中间的"空白区域"也包括进来了

---

### 第四步：为什么会出现这些问题？

CDF方法的本质缺陷：
1. 机械累积：无论概率高低，只要CDF没到0.9就继续累积
2. 忽略概率大小：把0.01和0.8的概率区间同等对待
3. 忽略连续性：不考虑概率分布是否连续
4. 无法自适应：不能根据分布特性调整策略

```
CDF的盲目累积过程：

0.01 + 0.02 + 0.01 + 0.03 + ... + 0.02 = 0.9 ✓ 达到目标
  ↑      ↑      ↑      ↑         ↑
 桶1    桶2    桶3    桶4  ...   桶20

问题：这些桶的概率都很低，说明模型根本不确定，
     但为了凑够90%，不得不全部包含进来！
```

| --- | --- | --- | --- | --- |
| 0.5 | [0.002, 0.979, 0.018, 0.000] | 0.979 | 0 | 极度尖锐 |
| 1 | [0.041, 0.831, 0.112, 0.015] | 0.831 | 0.015 | 尖锐 |
| 2 | [0.129, 0.579, 0.213, 0.078] | 0.579 | 0.078 | 平缓 |

| [图片] | - | - | - | - |

关键观察：
- T < 1（如0.5）：概率差异被放大 → 分布更尖锐 → 预测区间更小 → 命中率可能降低

✓ 调节温度可以在命中率和区间宽度之间找到最佳平衡点！

## 第三部分：波峰合并法（Peak Merging Method）

现在我来详细讲解如何用波峰合并法替代CDF方法来解决之前的问题。

### 第一步：核心思想对比

| 对比维度 | CDF方法 | 波峰合并法 |
| --- | --- | --- |
| 核心思想 | 累积概率直到达到90% | 找到高概率峰并向外扩展 |
| 处理逻辑 | 从左到右机械累加 | 从峰值向两侧智能合并 |
| 关注重点 | 累积概率总和 | 峰值概率及其邻域 |
| 合并策略 | 无选择性，全部累加 | 只合并相对较高的邻近区间 |
| 优势 | 简单直接 | 自适应、物理可解释 |
| 适用场景 | 仅适用于单峰尖锐分布 | 适用于各种分布（平缓、多峰等） |

---

### 第二步：算法的5个步骤

| 步骤 | 名称 | 具体操作 | 关键点 |
| --- | --- | --- | --- |
| 步骤1 | 识别最高峰 | 找到概率最大的桶（峰值） | 这是概率最集中的位置 |
| 步骤2 | 设定合并阈值 | 计算阈值 = 峰值概率 × α（如α=1/2, 1/3, 1/4） | α越小，阈值越低，合并越多桶 |
| 步骤3 | 向左合并 | 从峰值向左扩展，合并概率 ≥ 阈值的连续桶 | 遇到低于阈值的桶时停止 |
| 步骤4 | 向右合并 | 从峰值向右扩展，合并概率 ≥ 阈值的连续桶 | 遇到低于阈值的桶时停止 |
| 步骤5 | 输出区间 | 合并后的连续区间作为最终预测 | 区间宽度自适应调整 |

---

### 第三步：阈值参数α的影响

| 参数α | 阈值公式 | 合并倾向 | 区间宽度 | 命中率预期 |
| --- | --- | --- | --- | --- |
| 1月2日 | 阈值 = 峰值 × 0.5 | 保守：只合并高概率桶 | 较窄 | 可能较低 |
| 1月3日 | 阈值 = 峰值 × 0.33 | 中等：合并较高概率桶 | 中等 | 适中 |
| 1月4日 | 阈值 = 峰值 × 0.25 | 激进：合并更多桶 | 较宽 | 可能较高 |
| 自适应选择 | 根据实验结果选择最优α | - | 平衡最优 | 需要实验验证 |

## 补充：主线落地方案（与学长口径对齐）

### 1) 口径分工（先分清什么用 greedy，什么用 beam）
- **分桶回归（单点命中率）**：用 greedy  
  - 目标：找最大概率桶 → `bin_acc_0.2 / bin_acc_0.4`  
  - 这是“单点最大化”的任务，用 greedy 合理
- **区间预测（分布 + 区间）**：用 beam  
  - 目标：用完整分布输出区间 → `PICP / MPIW / width_stratified_PICP`  
  - greedy 只保留单路径，分布会坍缩，不适合区间口径

### 2) 波峰合并的落地实现（代码层思路）
**输入**：每个样本的 `leaf_probs`（来自 beam/full），以及 bin 边界  
**输出**：区间 `[L, U]`

**建议实现步骤（单峰版本，先做最稳的）**
1. 找最大峰：`peak_idx = argmax(leaf_probs)`  
2. 设阈值：`threshold = leaf_probs[peak_idx] * alpha`  
3. 向左扩展：只合并“连续且 prob >= threshold”的桶  
4. 向右扩展：同上  
5. 取区间边界：`L = bin_edges[left_idx]`, `U = bin_edges[right_idx + 1]`  

**关键细节**
- 用 **bin 边界**（不是桶中心）作为 L/U，避免区间系统性偏窄  
- 保证连续性：一旦遇到低于阈值的桶就停止  
- 先做单峰版本，避免多峰并集导致“非连续区间”

### 3) 参数设置（先固定，再做扫参）
- `alpha` 建议从 `0.5 / 0.33 / 0.25` 开始  
- `temperature` 先固定 `T=1`，避免和 alpha 叠加干扰  
- beam 主线：`K=16`，对照：`K=32`（full 只做诊断上限）

### 4) 最小可跑实验流程
1. **分桶回归基线**：greedy → `bin_acc_0.2 / bin_acc_0.4`  
2. **区间 CDF 基线**：beam K=16 → CDF 区间  
3. **波峰合并**：beam K=16 + peak_merge  
4. **对照**：beam K=32 + peak_merge  

### 5) 输出指标（与 0131 计划对齐）
- 主指标：`PICP / MPIW`  
- 诊断：`width_stratified_PICP`（看不同区间宽度的命中率）  
- 分桶回归：`bin_acc_0.2 / bin_acc_0.4`

## 第八步：波峰/波谷可视化与执行逻辑（先看清，再归并）

### 1) 波峰/波谷到底是什么？
把每个样本的 `leaf_probs` 看作一条“沿着 y 值的概率曲线”：
- **横轴**：bin 的数值区间（建议用 bin center / bin edge）  
- **纵轴**：该 bin 的概率（leaf_probs）  
- **波峰**：局部最大值（概率集中）  
- **波谷**：概率很低的连续区域（断层/空白）  

直觉：波峰代表“模型最相信的区域”，波谷代表“模型几乎不可能的区域”。  
波峰合并就是：从最高峰出发，只向两侧吃掉“还不算太低”的邻居，遇到波谷就停。

### 2) 为什么要先做可视化？
可视化不是必须步骤，但它能解决两个关键疑问：
1. **到底是单峰还是多峰？**（决定是否需要多峰策略）  
2. **阈值 alpha 该取多大？**（直接观察峰值附近概率衰减速度）  

如果曲线很平缓：CDF 会很宽，peak-merge 也会偏宽；  
如果曲线很尖：CDF 会很窄，但 greedy 可能过窄；  
用图能快速判断“分布到底长什么样”。

### 3) 可视化要画什么（建议最小三件套）
**单样本图（局部诊断）**
- 曲线：`leaf_probs`  
- 标注：`y_true`  
- 标注：`CDF 区间 [L_cdf, U_cdf]`  
- 标注：`Peak-merge 区间 [L_pm, U_pm]`  

**样本集合图（结构统计）**
- 峰数分布（peak count histogram）  
- 分布熵（entropy histogram）  
- 峰值概率分布（max prob histogram）  

**对比图（观察算法差异）**
- 同一批样本，CDF vs Peak-merge 的区间宽度直方图  
- 同一批样本，命中率 vs 宽度分桶表（已有 width_stratified_PICP）  

### 4) 执行逻辑（一步步做）
1. **固定分布生成口径**  
   - 主线：beam K=16  
   - 对照：beam K=32  
2. **导出 leaf_probs**  
   - 从 beam 输出的 leaf_probs 中抽样 100～500 条样本  
   - 保留 `bin_edges` 和 `y_true`  
3. **画单样本曲线**  
   - 看波峰/波谷的形状  
   - 先手动验证阈值 alpha 选 0.5/0.33/0.25 是否合理  
4. **实现 peak-merge**  
   - 用单峰版本（连续区间）  
   - 先不处理多峰  
5. **跑指标对比**  
   - CDF vs Peak-merge  
   - 指标见下表  

### 5) 区间预测要看哪些指标？
**必报（主线）**
- `PICP`：覆盖率  
- `MPIW`：平均宽度  
- `width_stratified_PICP`：分宽度命中率  

**建议补充（看分布是否“合理”）**
- **Entropy(mean)**：分布是否过尖/过平  
- **Peak count**：多峰比例（是否频繁出现“断层”）  
- **Max prob**：最高峰平均值（反映自信程度）  

这些不是替代 PICP/MPIW，而是解释“为什么某次区间很宽/很窄”。  
尤其是你们关心的“波峰/波谷”，必须通过这些分布特征来支撑结论。

### 6) 先看图还是先实现？
建议顺序：  
**先导出+可视化 50~100 个样本 → 直观看分布 → 再实现 peak-merge**。  
原因：这一步能直接指导 `alpha` 选取，并确认“是否存在明显断层”。

## 第九步：Greedy 软掩码（学长口径）— 仅用于诊断

### 1) 掩码到底是什么意思？
在 greedy 推理中，每一步都会决定“走左还是走右”，但模型在该步仍然会给**全体 bin**一个概率。  
学长说的“乘 mask”，就是**把当前不可达区间的概率缩小**，避免它们在后续 `leaf_probs = Π step_probs` 时产生不合理影响。  

直觉例子：  
- 第一步选择了“右半区间”  
- 那么“左半区间”的概率应该被削弱（不是完全归零），否则会形成与前缀不一致的概率质量  

### 2) 这和 ACM（训练 mask）不是一回事
- **ACM**：训练时降低负样本桶惩罚（loss 里加权）  
- **Greedy 软掩码**：评估时对“不可达区间”降权（推理时加权）  
两者层级不同，不要混用概念。

### 3) 我做了什么改动？
在 `scripts/eval.py` 的 `_greedy_step_probs` 里加了**可选 soft mask**：
- `mask_outside` ∈ [0, 1]  
- `mask_outside=1.0` 表示**不做掩码**（保持原逻辑）  
- `mask_outside=0.0` 等同**硬掩码**（不建议，会让分布坍缩）  
- 建议先用 `0.1 ~ 0.3` 观察效果  

### 4) 目的是什么？是否对齐主线？
- **目的**：测试“不可达区间权重是否影响 greedy 的分布形状”  
- **定位**：只做诊断，不是主线  
- **主线**仍然是 **beam K=16 的分布输出 + peak-merge**  
- 软掩码只是为了回答“学长提出的疑问：不相关节点是不是被过度影响”

### 5) 使用方法（示例）
```bash
PYTHONPATH=src python3 scripts/eval.py \
  --ckpt outputs/california_housing/run_20260202_112549 \
  --mode greedy \
  --temperature 1.0 \
  --mask-outside 0.1
```

### 6) 预期现象（判断是否有效）
- `mask_outside` 越小 → 概率越集中 → 区间更窄 → PICP 往下  
- `mask_outside` 越大 → 趋近原始 greedy  
- 这个趋势若成立，说明“mask 在控制分布坍缩程度”，但**它不能替代 beam**

## 第十步：多峰/波谷判定与合并策略选择（跨数据集异质性）

你们关心的“波峰/波谷”本质上是**分布是否多峰**。  
如果数据集天然双峰或多峰，单一连续区间无法同时覆盖多个峰，因此“是否合并、合并多少”必须**先判定分布形态**，再决定策略。

### 1) 先判定分布形态（可计算的指标）
对每个样本的 `leaf_probs` 计算以下诊断量：
- **peak_count**：局部峰数量（prominence ≥ τ·max_prob，建议 τ=0.2 起步）
- **top2_ratio**：第二峰 / 第一峰（反映多峰强度）
- **valley_ratio**：两峰之间最低谷 / 第一峰（反映断层深度）
- **entropy_norm**：归一化熵（反映分布平坦度）

解释直觉：
- peak_count > 1 且 top2_ratio 高 → 多峰样本
- valley_ratio 很低 → 峰之间有明显断层，不能跨越合并
- entropy_norm 高 → 分布平缓，区间需要更宽

### 2) 决策规则（合并 vs 不合并）
**规则 A：单峰或弱多峰 → 允许合并**
- 条件：`peak_count == 1` 或 `top2_ratio < 0.5` 或 `valley_ratio > 0.2`
- 策略：使用 **peak-merge**（阈值 = max_prob × alpha）

**规则 B：强多峰且断层明显 → 不跨谷合并**
- 条件：`top2_ratio ≥ 0.6` 且 `valley_ratio ≤ 0.1`
- 策略（二选一）：
  - **保守覆盖**：该样本回退 CDF（保证 PICP）
  - **保守精准**：只取最高峰区间（会降低 PICP，但更符合物理连续性）

### 3) 自适应 alpha（合并多少）
alpha 不必固定，可以按样本分布自适应：
- **峰更尖锐（entropy_norm 低）→ alpha 大 → 区间窄**
- **分布更平缓（entropy_norm 高）→ alpha 小 → 区间宽**

示例公式（可选）：
```
alpha_i = clip(alpha_min + (1 - entropy_norm) * (alpha_max - alpha_min), alpha_min, alpha_max)
```
建议起步：`alpha_min=0.2`, `alpha_max=0.5`

### 4) 主线评估口径（验证策略是否有效）
把验证集按“单峰 / 多峰”分组汇报：
- `PICP / MPIW`（每组）
- `width_stratified_PICP`（每组）

如果“多峰组 PICP 显著偏低”，就应该启用规则 B（回退 CDF 或提高温度）。

### 5) 推荐配套图与表（最小集）
- 峰数量直方图（peak_count）
- top2_ratio 分布
- valley_ratio 分布
- 单峰 vs 多峰的 PICP/MPIW 表

结论：  
**先用分布诊断判定“能不能合并”，再决定“合并多少”**，才能在多数据集异质性下稳定收敛。
